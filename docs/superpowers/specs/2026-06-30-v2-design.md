## 1. 概述

v2.0 包含三大模块改造：ApiAccessLog 日志优化、登录 SSO 改造、技改任务管理功能开发。

| 优先级 | 模块 | 理由 |
|--------|------|------|
| P0 | ApiAccessLog 日志优化 | 影响所有接口的问题排查，改动小收益大 |
| P1 | 登录 SSO 改造 | 用户认证基础，后续功能依赖用户信息 |
| P2 | 技改任务管理 | 业务功能，依赖前两者就绪 |

## 2. 模块一：ApiAccessLog 日志优化

### 2.1 改动范围

仅修改 `backend/techmg-framework/.../log/ApiAccessLogAspect.java`，不改表结构。

### 2.2 优化内容

**1) 结构化日志输出**

每条 API 日志改为固定格式：

```
[API] POST /api/xxx | IP: 10.x.x.x | UA: PostmanRuntime/7.x | Operator: admin
     Request: {"authNo":"xxx","password":"***"}
     Response(code=200, size=256B): {"code":200,"data":{...}}
     Cost: 23ms
```

**2) 补全字段**

- `User-Agent` — 从 `request.getHeader("User-Agent")` 获取
- `Referer` — 从 `request.getHeader("Referer")` 获取
- 完整 URL 含 query string — `request.getRequestURI()` + `request.getQueryString()`
- 响应长度 — `resultJson.length()` 字节数

**3) 可配置截断**

- 当前写死 2000 字符，改为 `application.yml` 中可配置
- 配置项：`api.access-log.max-length: 4000`（默认 4000）
- 截断时尾部标记 `...[truncated, total: XXXX chars]`

**4) 异常日志增强**

异常时记录堆栈摘要（前 3 行），而非仅 `e.getMessage()`：

```
[API] POST /api/xxx | ... | ERROR: NullPointerException
     Stack: at com.icbc.sh.techmg.controller.XxxController.create(XxxController.java:42)
            at java.base/jdk.internal.reflect.NativeMethodAccessorImpl.invoke0(Native Method)
            at ...
            ...[stack truncated]
     Cost: 156ms
```

### 2.3 不影响

- `OperationLogEvent` 及数据库写入逻辑不变
- `sys_operation_log` 表不变
- 控制器上的 `@ApiAccessLog` 注解用法不变

## 3. 模块二：登录 SSO 改造

### 3.1 认证流程

```
前端                                      后端                           行内SSO/API
 |                                         |                                |
 |-- GET /api/auth/sso/login-url -------->|                                |
 |<-- {loginUrl: "https://sso.icbc..."}---|                                |
 |                                         |                                |
 |-- 浏览器重定向到 SSO 登录 -------------|------------------------------->|
 |<-- SSO 回调 ?ticket=xxx ---------------|                                |
 |                                         |                                |
 |-- POST /api/auth/sso/login {ticket} -->|                                |
 |                                         |-- 验证 ticket --------------->|
 |                                         |<-- {userId: "xxx"} ------------|
 |                                         |                                |
 |                                         |-- 查询用户扩展信息 ----------->|
 |                                         |<-- {tellername,ad,branch...} --|
 |                                         |                                |
 |                                         |-- 写入/更新 sys_user 记录       |
 |                                         |-- 生成 JWT (含扩展信息)         |
 |<-- {token, userInfo: {tellername,ad,   |
 |     branchId,branchName,notesId,        |
 |     branchIdList, roles}}               |
 |                                         |                                |
 |-- 前端 AES 加密缓存 userInfo 到 localStorage |
```

### 3.2 数据库变更

#### sys_user 表 — 新增字段

```sql
ALTER TABLE sys_user ADD COLUMN ad_account VARCHAR(64) DEFAULT '' COMMENT 'AD账号';
ALTER TABLE sys_user ADD COLUMN branch_id VARCHAR(32) DEFAULT '' COMMENT '机构号';
ALTER TABLE sys_user ADD COLUMN branch_name VARCHAR(200) DEFAULT '' COMMENT '机构全名';
ALTER TABLE sys_user ADD COLUMN notes_id VARCHAR(64) DEFAULT '' COMMENT 'Notes ID';
ALTER TABLE sys_user ADD COLUMN last_login_time DATETIME DEFAULT NULL COMMENT '最后登录时间';
```

#### 新增：sys_user_branch 表

```sql
CREATE TABLE sys_user_branch (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL COMMENT '用户ID',
    branch_id VARCHAR(32) NOT NULL COMMENT '机构号',
    branch_name VARCHAR(200) DEFAULT '' COMMENT '机构名',
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户多机构关联表';
```

#### 废弃但不删除的字段

`password`, `email`, `phone` 字段保留在表中但不再赋值使用。

### 3.3 sys_user 表职责重新定义

| 字段 | 来源 | 用途 |
|------|------|------|
| auth_no | SSO 登录 | 用户唯一标识 |
| real_name (tellername) | 内部 API | 显示中文名 |
| ad_account | 内部 API | AD 账号 |
| branch_id / branch_name | 内部 API | 机构筛选 |
| notes_id | 内部 API | 备注 |
| last_login_time | 系统自动记录 | 使用时间追踪 |
| status | 管理员设置 | 启用/禁用 |
| 角色关联 (sys_user_role) | 管理员分配 | 本地授权 |

核心定位：**平台使用记录追踪 + 本地授权**。

### 3.4 前端缓存方案

- 登录后 `userInfo` 用 AES 加密存 `localStorage`
- AES 密钥从环境变量 `VITE_USERINFO_ENCRYPT_KEY` 注入
- Pinia store 内存放解密后的明文
- 页面顶部显示中文名 + 机构名
- 列表页可按 `branchId` 筛选数据

### 3.5 向后兼容

- 原 `POST /api/auth/login` (authNo + password) 保留兼容
- `SsoAuthProvider` 接口实现具体 SSO 逻辑
- 默认管理员 `admin` 本地密码登录保留

### 3.6 新增 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/auth/sso/login-url` | 获取 SSO 登录地址 |
| POST | `/api/auth/sso/login` | SSO ticket 登录 |
| POST | `/api/auth/logout` | 退出（复用，增加 SSO 登出回调） |

## 6. 已确认的设计决策

| 决策 | 结论 |
|------|------|
| SSO 接入模式 | SSO 重定向模式 (ticket → 验证 → 查询扩展信息 → JWT) |
| sys_user 表定位 | 平台使用记录追踪 + 本地授权，去除密码依赖 |
| 任务大类/小类 | 数据字典可配置 |
| 任务来源 | 数据字典 + 支持手动输入 |
| 状态流转 | PENDING → IN_PROGRESS → COMPLETED → CLOSED |
