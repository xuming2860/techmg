## 1. 项目背景

工商银行上海研发基地内部技术管理平台，面向研发部提供技术治理、技术整改、资产管理等内容的线上化管理。

### 1.1 项目目标

- 替代线下 Excel 驱动的治理流程，实现线上化闭环管理
- 统一管理基地内各应用的技术资产
- 提供多角色权限体系，支撑不同层级用户使用
- 统一认证号登录 (auth_no)，无需注册，角色赋权

## 4. 角色权限体系

### 4.1 5 种角色

| 角色 | role_code | 权限范围 |
|------|-----------|----------|
| 平台管理员 | PLATFORM_ADMIN | 全局管理：所有用户/角色/权限/菜单/部门配置，可见全部26个菜单 |
| 部门管理员 | DEPT_ADMIN | 本部门用户管理、资产管理，可见技术管理+资产管理+生产管理+数智生态 |
| 部门DBA | DEPT_DBA | 数据库治理/巡检任务操作、数据库参数管理，可见数据库管理+资产管理+数智生态 |
| 普通用户 | NORMAL_USER | 填写反馈、确认治理任务、查看本部门数据，可见技改任务+巡检任务 |
| 游客 | GUEST | 仅查看公开报表、治理情况概览（只读），仅可见首页 |

### 4.2 权限控制

- 路由守卫：beforeEach 检查 JWT token
- 动态路由：后端按角色过滤菜单 → 前端 addRoute 注册
- 按钮级权限：v-permission 指令 (已注册，待页面中使用)
- 方法级安全：@EnableMethodSecurity 已启用，待添加 @PreAuthorize

## 5. 安全认证

### 5.1 认证流程

1. 前端提交 authNo + password
2. 后端 BCrypt 验证密码
3. 返回 JWT token (24h 有效期) + userInfo + roles
4. 前端存储 token 到 localStorage + Pinia
5. 每次请求 Axios 拦截器自动注入 `Authorization: Bearer {token}`
6. 后端 JwtAuthenticationFilter 验证 token

### 5.2 安全特性

- Spring Security 6 无状态会话
- BCrypt 密码加密
- JWT Bearer token 认证
- SSO 扩展接口预留 (SsoAuthProvider)
- 幂等性支持 (@Idempotent 注解 + Redis)
- 登录防重复提交 (1分钟内幂等)
- DataInitializer 启动自动创建默认管理员 (admin/admin123)

## 6. 后端架构

### 6.1 模块结构

```
backend/
├── pom.xml                     # 父POM (版本号/依赖管理)
├── techmg-common/              # 公共底座 (零业务依赖)
├── techmg-framework/           # 框架层 (横切能力)
├── techmg-system/              # 系统管理 (RBAC)
├── techmg-business-api/        # 业务接口定义
├── techmg-business-service/    # 业务接口实现
└── techmg-admin/               # 启动模块 (Spring Boot入口)
```

### 6.2 核心约束

1. 配置文件 .yml 格式，禁止 .properties
2. 简单CRUD用API，复杂SQL用XML Mapper，禁止 @Select/@Update 注解SQL
3. Gson extendMessageConverters (保留 String 转换器)
4. 所有 API 返回 R\<T\> 统一响应
5. 抛 BusinessException → GlobalExceptionHandler 统一拦截
6. @ApiAccessLog 注解 → Spring Event → 写入操作日志表
7. @DS("master") 默认, @DS("slave") 从库读写分离
8. 逻辑删除 deleted (0正常/1删除), @TableLogic
9. 自动填充 createTime/createBy/updateTime/updateBy
10. Log4j2 2.17.1, 排除 Logback, 100MB/30天滚动
11. Security 无状态JWT, /api/auth/** 放行
12. maven-compiler-plugin 必须 -parameters
13. Gson LocalDateTime 自定义序列化 (yyyy-MM-dd HH:mm:ss)
14. Spring Event 跨模块通信 (framework 发布 → system 监听, 参考 OperationLogEvent)
15. Redis 缓存键统一常量 (CacheConstants: token/perm/menu/dict/excel)
16. 公共枚举 (GovernanceStatus/DbTypeEnum/InspectionTypeEnum) + TreeUtil 泛型树形构建器

### 7.2 核心设计

- **API 层**：页面禁止直接 axios，必须调用 api 层具名导出函数
- **请求封装**：Bearer token 注入 + R\<T\> 解包 + Blob 直通 + 错误分类
- **动态路由**：
  - 登录：`login()` → `getUserMenuTree()` → `generateRoutes()` → `router.addRoute('main', route)`
  - 刷新：`beforeEach` 检测 `routesLoaded=false` → `loadRoutes()` → API → `addRoute` → `next({...to, replace:true})` 重新导航
  - 404 兜底：`generateRoutes()` 最后注册 `/:pathMatch(.*)*` → `views/error/404.vue` (渐变蓝色 404 页面)
- **布局**：`route.meta.layout` 决定上左右/上下，首页 wide 布局 (后端 `SysMenu.layout` 字段: id=1→top, 其余→side)
- **TopBar**：ICBC 蓝色 logo 标 + 模块 tabs (递归 `getFirstPath` 找首叶子路径) + 圆形首字头像 + 下拉退出
- **Sidebar**：按活跃模块动态显示子菜单 (递归 MenuItem)、浅灰主题 (#fafbfc)、蓝色激活态 (#3370ff)
- **权限**：路由守卫 + 菜单按角色过滤 (`/api/system/menu/user-tree`) + v-permission 指令 (已注册)
- **Error Boundary**：`onErrorCaptured` 兜底，白屏变友好错误页 + 重试按钮
- **代码规范**：ESLint + Prettier 统一风格，`npm run lint`

### 9.4 MySQL 连接

```bash
# Docker MySQL 必须配置字符集 (否则中文乱码)
# 在 /etc/mysql/conf.d/charset.cnf 中配置:
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_general_ci

# 连接时必须指定 --default-character-set=utf8mb4
docker exec -i mysql mysql -uroot -p --default-character-set=utf8mb4 tmvp

# JDBC URL (必须 allowPublicKeyRetrieval)
jdbc:mysql://localhost:3306/tmvp?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true

# Druid connection-init-sqls 确保连接字符集
spring.datasource.druid.connection-init-sqls: SET NAMES utf8mb4
```
